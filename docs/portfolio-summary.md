# Portfolio Summary

## 프로젝트 소개

FinRecon은 결제 거래 검증, 내부 원장 대사, 일일 정산을 처리하는 백엔드 포트폴리오 프로젝트다. 파일 업로드와 검증, 금액 계산, 재실행 가능한 업무 처리, 인증/권한, 감사 로그를 포함한다.

## 보여주는 역량

- Spring Boot 기반 REST API 설계와 계층형 구조
- JPA/Flyway 기반 명시적 데이터 모델링
- CSV 파일 처리와 행 단위 오류 분리
- 거래 금액 정합성 검증과 중복 방지
- 대사 결과 분류와 일자별 재처리
- BigDecimal 기반 정산 계산
- JWT 인증과 역할 기반 접근 제어
- 감사 로그와 실행 이력으로 운영 추적성 확보
- MockMvc 통합 테스트와 CI 친화적 H2 테스트 구성

## 핵심 시나리오

1. 운영자가 로그인한다.
2. 외부 거래 CSV를 업로드한다.
3. 내부 원장 CSV를 업로드한다.
4. 업무 일자 기준으로 대사를 실행한다.
5. 대사 결과가 `MATCHED`인 거래를 기준으로 정산한다.
6. 관리자가 감사 로그와 실행 이력을 확인한다.

## 설계 포인트

- 오류 행을 별도 테이블에 저장해 전체 파일 재처리 없이 수정 대상을 파악할 수 있다.
- 같은 업무 일자의 대사/정산은 재실행 가능하도록 기존 결과를 교체한다.
- 금액은 부동소수점이 아니라 `BigDecimal`과 `decimal(19,2)`를 사용한다.
- 업무 API는 JWT로 보호하고 감사 로그 조회는 ADMIN으로 제한한다.
